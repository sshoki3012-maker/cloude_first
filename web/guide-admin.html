未来予想 投票アプリ
準備と当日運営の進め方

画面役割
/index.html投票者の入口（合言葉→名前選択）
/vote.html投票（お題ごと最大3人・保存/変更）
/results.html結果スクリーン（プロジェクタ・自動更新）
/admin.html管理（受付・集計・編集・リセット）
/qr.html投票URLのQR表示

構成：静的サイト（Netlify）＋ Supabase。矢印キー（← →）で進みます。

STEP 1 / 事前準備（前日まで）

DB作成 → 設定 → 名簿・お題の登録

  1. Supabaseで supabase/schema.sql → seed.sql をSQL Editorで実行
  2. web/config.js に URL・Publishable key・合言葉を設定
    投票者用 VOTER_PASSCODE と 管理用 ADMIN_PASSCODE は別々
  3. Netlifyにデプロイ（mainに連携すると自動公開）
  4. admin画面で名簿（約40名）とお題を本番用に差し替え
お題は後から追加・編集・表示ON/OFF可。テスト投票は当日前に 「投票をリセット」で消しておきます。

STEP 2 / 当日の運営フロー

配布 → 投票 → 締切 → 発表

  1. qr.html をプロジェクタに映し、QRと合言葉を案内
  2. 参加者がスマホで投票（管理はadmin画面で投票状況を確認）
  3. 頃合いで admin の「受付を締め切る」→ 以降は変更不可に
  4. results.html を投影して結果発表（数秒ごと自動更新・お題切替）
  5. 結果をエクスポートして保存（次回の答え合わせ用）
結果画面はポイント順（傾斜配点）で上位を大きく表示。 「お題を自動で切替」にチェックすると一定間隔で巡回します。

STEP 3 / 管理画面（admin.html）の操作

合言葉で解除 → 各操作

入口で ADMIN_PASSCODE を入力して解除します。

受付の切り替え

「受付中／締切」をワンタップで切替。締切後は投票画面がロック。

投票状況

「40人中○人が投票」「全○問回答○人」を集計表示。「更新」で最新化。

お題・名簿の編集

その場で追加・改名・削除・表示ON/OFF。

リセット

この回の投票を全削除（テスト用）。本番開始前のクリアに使用。

データ保存・答え合わせ・注意点

予想は残す。次回につなぐ

エクスポート：admin から JSON / CSV で全投票を保存。 event_id（年度）付きで記録され、次回は新しい年度を作れば過去データは残ります。

セキュリティの割り切り：合言葉は公開ファイル（config.js）に入る 「余興レベルの軽い保護」です。当日URLは身内だけに配布してください。 厳密にするなら Supabase Auth 化が必要です。

困ったら：締切は admin の受付トグル／集計は「更新」ボタン／ やり直しは「リセット」。これだけ押さえれば運営できます。
